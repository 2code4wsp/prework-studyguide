HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Defines the structure and content of the website
  • made up of elements such as text, image, header and list elements
  • uses elements that divide the content on the page and describe this content such as meta-related, content division, and semantics
  • it is the first element used in a website, much like the framework of a building
  • metadata is the element containing information
  • the body contains content shown to the user
  • HTML attributes are always defined in the beginning element, never the closing
  • Metadata is data about our webpage that the browser uses to serve our webpage to users.
  • Metadata contains accessibility, search engines and performance
  • metadata tags are required and extremely important for SEO
  • SEO is search engine optimization

CSS

  • ENTER your CSS notes here
  • CSS stands for cascading style sheets and defines how webpage content is presented
  • there are 3 ways to style a webpage: inline CSS, internal CSS, and external CSS
  • inline styles the CSS directly in HTML file (useful for quick permanent changes
  • internal styles are embeded in HTML using style element in the head element. A single change to the CSS rule will apply to all tagged page elements
  • external CSS are most common. Allows developers to keep CSS rules separate in files making design changes easier
  • external style sheets link the file using link element placed in HTML's head
  • separation of concerns refers to separating code into different files based on use

Git

  • Enter your Git notes here
  • ls : lists the contents of the current directory
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • cp: creates a copy
  • pwd: print working directory: we check our working directory and it shows us the path
  • cd document or desktop: navigates to this directory
  • touch index.html : we use touch to create a new file named index.html
  • ls: is used to check that a new file has been successfully added
  • cd .. : is used to move back up to the previous directory
  • git clone will create a duplicate of a repository/branch
  • mkdir will make a new directory
  • git add -A : adds changes (such as touch)
  • git commit -m : commits our changes and add a message of the changes made in quotations
  • git push : will push our changes back up to the repo (in github)
  • git pull : to pull changes from the repo created on github and update the repo on local machine
  • on git pull origin main, the origin refers the the repo that was cloned and main is the branch. The default branch is main
  • when you make a new directory and clone it, the "cloning into" will then be used in a cd to make sure you are on your "main"
  • git fetch :command that brings my local copy of remote repository up to date
  • git pull: brings the changes in the remote repository to where I keep my own code
